# এই নোটটিতে আমরা SQL-এর চারটি গুরুত্বপূর্ণ কমান্ড: CREATE TABLE, ALTER TABLE, DROP TABLE এবং INSERT INTO TABLE সম্পর্কে বিস্তারিত শিখব।

**SQL টেবিল কমান্ডসমূহ:** বিস্তারিত গাইড
একজন সফটওয়্যার ইঞ্জিনিয়ার হিসেবে ডেটাবেস ম্যানেজমেন্টের জন্য এই চারটি কমান্ড জানা আপনার জন্য অপরিহার্য। নিচে প্রতিটি কমান্ডের কাজ এবং ব্যবহার ধাপে ধাপে ব্যাখ্যা করা হলো।

## ১. `CREATE TABLE` (টেবিল তৈরি করা): ডেটাবেসের ভেতরে নতুন একটি টেবিল তৈরি করার জন্য `CREATE TABLE` কমান্ড ব্যবহার করা হয়। 

**সিনট্যাক্স (Syntax):**

```sql
CREATE TABLE table_name (
	column1 datatype constraint,
    column2 datatype constraint,
    ...
    table_constraints
);
```

**কোড এবং ব্যাখ্যা:**

**এইখানে একটি `employees` টেবিল তৈরি করার উদাহরণ দেওয়া হয়েছে:**

```sql
CREATE TABLE employees (
    employee_id INT PRIMARY KEY AUTO_INCREMENT,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE,
    higher_date DATE DEFAULT (CURRENT_DATE),
    salary DECIMAL(10, 2) CHECK (salary > 0),
    employment_status ENUM('active', 'leave', 'terminated') DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**লাইন-বাই-লাইন বিশ্লেষণ:**

- `CREATE TABLE employees`: এটি দিয়ে `employees` নামে একটি নতুন টেবিল তৈরির নির্দেশ দেওয়া হচ্ছে।

- `( এবং )`: এই প্রথম বন্ধনীর ভেতরে সব কলামের বিবরণ থাকে।

- `employee_id INT`: `employee_id` হলো কলামের নাম এবং `INT` (Integer) মানে এখানে শুধু পূর্ণসংখ্যা বসবে।

- `PRIMARY KEY`: এটি একটি Constraint বা নিয়ম। এর মানে হলো এই আইডিটি প্রতিটি কর্মীর জন্য আলাদা (Unique) হবে এবং এটি খালি রাখা যাবে না।

- `AUTO_INCREMENT`: এটি একটি প্রপার্টি। যদি আপনি আইডি না দেন, তবে ডেটাবেস নিজে থেকেই আগের আইডির সাথে ১ যোগ করে নতুন আইডি তৈরি করে নেবে।

- `VARCHAR(50)`: এটি দিয়ে টেক্সট বোঝায়। `50` মানে সর্বোচ্চ ৫০টি অক্ষর রাখা যাবে।

- `NOT NULL`: এই নিয়মের ফলে আপনি `first_name` বা `last_name` খালি রেখে ডেটা সেভ করতে পারবেন না।

- `UNIQUE`: যেমন `email`; একই ইমেইল দুইজন ব্যবহার করতে পারবে না।

- `DECIMAL(10, 2)`: এর মানে মোট ১০টি সংখ্যা থাকবে যার মধ্যে ২ ঘর থাকবে দশমিকের পরে (যেমন: ১২৩৪৫৬৭৮.৯০)।

- `CHECK (salary > 0)`: এটি একটি শর্ত। বেতন অবশ্যই ০-এর বেশি হতে হবে, না হলে ডেটা গ্রহণ করবে না।

- `ENUM`: এটি একটি লিস্ট। যেমন `employment_status`-এ শুধু `'active', 'leave'` অথবা `'terminated'`—এই তিনটি মানের যেকোনো একটি থাকতে পারবে।

- `DEFAULT`: যদি কোনো কলামে ডেটা না দেওয়া হয়, তবে এই মানটি স্বয়ংক্রিয়ভাবে বসে যাবে।

- `;`: সেমিকোলন দিয়ে SQL স্টেটমেন্ট শেষ করা হয়।

## ২. `INSERT INTO` (ডেটা প্রবেশ করানো)
টেবিল তৈরি হওয়ার পর তাতে তথ্য বা রো (row) যোগ করতে এই কমান্ড ব্যবহার করা হয়।

**কোড উদাহরণ:**

```sql
INSERT INTO employees (first_name, last_name, email, salary)
VALUES ('John', 'Smith', 'john@example.com', 50000.00);
```

**ব্যাখ্যা:**

- `INSERT INTO employees`: কোন টেবিলে ডেটা যাবে তা নির্দিষ্ট করা।

- `(first_name, ...)`: কোন কোন কলামে ডেটা দেবেন তার লিস্ট।

- `VALUES (...)`: কলামের ক্রমানুসারে প্রকৃত তথ্য প্রদান করা।

- যদি কোনো কলামে `DEFAULT` মান থাকে এবং আপনি সেটি `INSERT`-এ না লেখেন, তবে ডেটাবেস নিজে থেকেই ডিফল্ট মান বসিয়ে দেয়।

## ৩. `ALTER TABLE` (টেবিল পরিবর্তন করা)
টেবিল তৈরির পর তার স্ট্রাকচার (যেমন: নতুন কলাম যোগ করা বা কলামের নাম পরিবর্তন) বদলাতে এটি ব্যবহৃত হয়।

### ক) নতুন কলাম যোগ করা (Add Column):

```sql
ALTER TABLE employees 
ADD COLUMN description TEXT;
```

- `ADD COLUMN`: নতুন একটি ক্ষেত্র বা কলাম তৈরি করে।

---

### খ) কলাম পরিবর্তন করা (Modify/Rename Column):

```sql
ALTER TABLE employees 
MODIFY COLUMN department_id INT NOT NULL;
```

- `MODIFY`: কলামের ডেটা টাইপ বা কনস্ট্রেইন্ট পরিবর্তন করতে ব্যবহৃত হয়।

- `RENAME COLUMN old_name TO new_name`: কলামের নাম বদলাতে ব্যবহৃত হয়।

---

### গ) কলাম মুছে ফেলা (Drop Column):

```sql
ALTER TABLE employees 
DROP COLUMN description;
```

- `DROP COLUMN`: টেবিল থেকে কোনো নির্দিষ্ট কলাম চিরতরে সরিয়ে দেয়।

---

### ঘ) ফরেন কি যোগ করা (Add Foreign Key):
দুটি টেবিলের মধ্যে সম্পর্ক তৈরি করতে এটি ব্যবহৃত হয়।

```sql
ALTER TABLE employees 
ADD FOREIGN KEY (department_id) REFERENCES departments(department_id);
```

- এটি নিশ্চিত করে যে `employees` টেবিলের `department_id` অবশ্যই `departments` টেবিলে থাকতে হবে।

## ৪. `DROP TABLE` (টেবিল মুছে ফেলা)

পুরো টেবিল এবং তার ভেতরের সব ডেটা মুছে ফেলার জন্য এই কমান্ড ব্যবহৃত হয়।

```sql
DROP TABLE IF EXISTS employees;
```

- `DROP TABLE`: টেবিল ডিলিট করে।

- `IF EXISTS`: এটি একটি সুরক্ষা কবজ। টেবিলটি যদি না থাকে, তবে এরর না দেখিয়ে শুধু একটি ওয়ার্নিং দেবে।

- সতর্কতা: যদি একটি টেবিলের সাথে অন্য টেবিলের সম্পর্ক (Foreign Key) থাকে, তবে মূল টেবিলটি আগে ডিলিট করা যাবে না।

## Cheat Sheet

### ১. Basic Syntax

|   |   |   |
|---|---|---|
|Command|Purpose|Example|
|**CREATE**|নতুন টেবিল তৈরি|`CREATE TABLE users (id INT, name VARCHAR(20));`|
|**INSERT**|ডেটা যোগ করা|`INSERT INTO users (id, name) VALUES (1, 'Ali');`|
|**ALTER**|স্ট্রাকচার বদলানো|`ALTER TABLE users ADD age INT;`|
|**DROP**|টেবিল মোছা|`DROP TABLE users;`|

### ২. Important Constraints (নিয়মাবলী)

• `NOT NULL`: খালি রাখা যাবে না।

• `UNIQUE`: মানগুলো সবসময় আলাদা হতে হবে।

• `PRIMARY KEY`: ইউনিক আইডি এবং নট নাল-এর সমষ্টি।

• `DEFAULT`: মান না দিলে স্বয়ংক্রিয়ভাবে যা বসবে।

• `CHECK`: কোনো নির্দিষ্ট শর্ত যাচাই করা।

• `AUTO_INCREMENT`: নিজে নিজে মান বাড়িয়ে নেওয়া।

### ৩. Quick Examples for Alter Table

• **Add Column:** `ALTER TABLE table_name ADD column_name datatype;`

• **Modify Column:** `ALTER TABLE table_name MODIFY COLUMN col_name new_datatype;`

• **Drop Column:** `ALTER TABLE table_name DROP COLUMN col_name;`

• **Rename Column:** `ALTER TABLE table_name RENAME COLUMN old TO new;`

---

**মনে রাখবেন:** SQL-এ কমান্ডগুলো বড় হাতের (Capital) বা ছোট হাতের অক্ষরে লেখা যায়, তবে বড় হাতের অক্ষরে লেখা একটি ভালো প্র্যাকটিস। কোনো রো আপডেট করার সময় যদি ফরেন কি কনস্ট্রেইন্ট থাকে, তবে ডেটাবেস ভুল ডেটা ইনপুট নিতে বাধা দেবে, যা ডেটার সঠিকতা বজায় রাখে।
